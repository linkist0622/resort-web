# SHININGresort Webサイト開発｜プロジェクト前提ファイル v1.2

## 1. プロジェクト概要

- プロジェクト名：SHININGresort Webサイト開発（KOKON再現フェーズ → オリジナル構築）
- 開発者：LiNKiST
  - ITリテラシーはあるが、**Webアプリ開発・コーディングは初心者**
  - 目的は「**AIを相棒にしながらWeb開発スキルとプロンプト力を身につける**」こと
- 目的：
  - 開発初心者が **AI をフル活用してモダンな Web サイトを構築する経験を積む**。
  - 参考サイト **「宿-SHUKU- 古今」トップページ（https://shuku-kokon.com）** を、
    レイアウト・構成・UI/UX・アニメーションの観点で **可能な限り完全再現** する。
  - 特に、KOKON の
    - スクロールに応じたセクションの見せ方
    - フェードインやパララックスなどのアニメーション
    - 余白や行間、タイポグラフィも含めた UI / UX の心地良さ
    を学び取り、自社サイトの設計に活かす。
  - その過程で得たノウハウを活かし、
    **SHININGresort オリジナルサイト**としてブラッシュアップしていく。

- 大前提：
  - **中間ゴール**：KOKON トップページを Next.js によって  
    **「レイアウト＋アニメーション＋スクロール挙動」まで含めて再現**する。
  - **最終ゴール**：テキスト・写真・色・構成・動き（モーションデザイン）を
    SHININGresort に最適化した **完全オリジナルサイト**を構築する。

- 著作権・倫理的前提：
  - 参考サイトから **テキスト・画像・ロゴ・ブランド要素をコピーして公開しない**。
  - 「再現」はあくまで
    - レイアウト
    - コンポーネント構造
    - ページ構成
    - 余白・見せ方の思想
    - アニメーション・スクロール挙動の設計
    を学ぶことが目的。
  - 本番公開前に、  
    - テキスト → SHININGresortオリジナル文言  
    - 画像 → 自社で用意した写真 or 正規ライセンス素材  
    に必ず差し替える。

## 2. 開発者前提 & 学習方針

- LiNKiST は **ITには慣れているが、コードを書くのは初心者**という前提で進める。
- このプロジェクトでは、
  - 「黒魔術的に動くコード」ではなく、
  - **なぜその書き方なのか／何をしているのか** が分かることを重視する。

### 2-1. 手順と解説メモのスタイル

- 実装手順は、今後のチャットで
  - `STEP 1: 〜する`
  - `STEP 2: 〜する`
  のように**ステップごとに区切り**、
- 各ステップには必ず
  - **「なぜやるのか」解説メモ**
  - 必要に応じて **AI に投げるプロンプト例**
  をセットで記載する。

例）  
- 手順：`src/app/page.tsx` に HeroSection を読み込む。  
- 解説メモ：  
  - 「トップページの骨組みを作る場所はここ」  
  - 「細かい見た目は components 配下のコンポーネント側で調整」  
- AI プロンプト例：  
  - 「KOKONトップのヒーローセクションのように、画面いっぱいの画像とキャッチコピーを表示するコンポーネントを書いて」

### 2-2. AI活用・プロンプト学習の方針

- コードを書くときは、できるだけ **「やりたいことベース＋見た目のイメージ」**でAIに質問する。
- 例えば：
  - 「KOKONのトップページの○○セクションっぽく」
  - 「モバイルでは1カラム、PCでは2カラムに」
  - 「スクロールしたときにふわっとフェードイン」
- 本プロジェクトのドキュメント内では、
  - ときどき「**このタイミングでAIにこう聞くと良い**」という  
    **プロンプト例**を明示する。

## 3. 技術スタック（確定）

### フロントエンド

- Next.js 15（App Router 使用・`create-next-app` の最新安定版を利用）
- TypeScript
- React 18
- Tailwind CSS
- Framer Motion
  - KOKON に近いスクロールアニメーションやフェードインを実現するための主要ライブラリとする。

### インフラ / デプロイ

- Vercel
  - GitHub 連携で自動デプロイ
  - プレビュー環境を活用してデザイン＆アニメーションを確認

### データ / コンテンツ管理（フェーズ分け）

- Phase 1（KOKON再現フェーズ）
  - トップページのコンテンツは **コード内の定数 / JSON** として管理。
  - 例：`src/data/rooms.ts`, `src/data/experiences.ts`, `src/data/news.ts` など。

- Phase 2（オリジナル構築〜運用フェーズ）
  - Supabase（PostgreSQL）を採用し、  
    `news` や `rooms` などを DB 化。
  - 管理画面は後から検討（最初は Supabase コンソールから手入力でもOK）。

## 4. Next.js プロジェクト初期セットアップ

### 4-1. 前提環境

- Node.js：推奨 20 系
- パッケージマネージャ：`npm`（もしくは `pnpm` / `yarn`。本プロジェクトでは `npm` を想定）
- エディタ：VSCode または Cursor 推奨
- GitHub リポジトリを作成し、Git でバージョン管理

### 4-2. プロジェクト作成コマンド（例）

```bash
npx create-next-app@latest shiningresort-web   --typescript   --tailwind   --eslint   --src-dir   --app   --import-alias "@/*"
```

- プロジェクト名：`shiningresort-web`
- 主なオプション方針：
  - `--typescript`：最初から TypeScript で始める
  - `--tailwind`：スタイルは Tailwind で統一
  - `--src-dir`：`src` ディレクトリ配下にコードをまとめる
  - `--app`：App Router を使用
  - `--import-alias "@/*"`：`@/components/...` のような import を利用

### 4-3. 初期セットアップ後に行う調整（方針）

- `src/app/page.tsx` を **KOKON トップページ再現用のメインファイル**と位置づける。
- `globals.css` に Tailwind のベース設定を維持しつつ、
  - ベースフォント
  - 行間
  - 背景色
  などを調整し、KOKONの雰囲気に近づける。
- `Framer Motion` を導入し、
  - セクション単位のフェードイン
  - スクロール位置に応じたトランジション
  を少しずつ付与していく。
- 画像類は当初は `public/` に仮画像を配置し、  
  後から SHININGresort 用の写真に差し替える。

## 6. トップページ構成（セクション単位の前提）

※ ここでは**構造のみ**を記述し、テキスト本文はオリジナル作成を前提とする。

各セクションでは、**KOKON の挙動を参考にしつつ、以下のようなアニメーション方針**をとる。

1. **ヘッダー / グローバルナビ**
   - ロゴ
   - ナビ項目
     - 宿について
     - ご宿泊
     - お食事
     - 体験・散策
     - お知らせ
     - アクセス
   - 予約ボタン
   - スクロール時に
     - 高さが少し変化
     - 背景色がうっすら出る
     など、KOKONの落ち着いた挙動を参考にする。
   - 言語切り替え（JA / EN） → 実装は後フェーズで検討

2. **ヒーローセクション**
   - 大きなキービジュアル画像
   - キャッチコピー（日本語・英語）
   - スクロール誘導（矢印や「scroll」テキスト）
   - ロード時のフェードイン／ズームインなどを Framer Motion で実装。

3. **コンセプトセクション**
   - テキスト中心
   - 写真もしくは背景画像を組み合わせる
   - スクロールでゆっくりフェードインする挙動を付与。

4. **お知らせ（News）セクション**
   - 最新 3 件程度をカード形式で表示
   - カードはスクロールイン時にふわっと浮き上がるようなアニメーション。
   - 「一覧はこちら」へのリンク（`/news`）

5. **ご宿泊（Rooms）セクション**
   - コンセプトテキスト
   - 部屋のイメージ画像（スライダー or グリッド）
   - 画像が画面中央付近に来たときにフェードイン＋わずかなモーション。
   - 「部屋の一覧を見る」ボタン

6. **お食事（Restaurant）セクション**
   - コンセプトテキスト
   - メインビジュアル画像
   - テキストと画像が左右からゆっくり現れる挙動を検討。
   - 「くわしく見る」リンク（`/restaurant`）

7. **体験・散策（Experience）セクション**
   - 周辺の体験一覧風のブロック（画像＋ラベル）
   - 縦横の組み合わせレイアウト
   - スクロールに合わせて、グリッドのカードが順にフェードイン。
   - 「くわしく見る」リンク（`/experience`）

8. **アクセス（Access）セクション**
   - 宿名
   - 住所
   - アクセス方法（電車 / 車）
   - チェックイン・チェックアウト
   - 駐車場・備考
   - Google Maps へのリンク
   - テキストは静的でもよいが、セクション全体のフェードインは検討。

9. **予約セクション**
   - 宿泊予約ボタン
   - 電話番号・受付時間
   - CTA（行動喚起）として、ホバー時のアニメーションはやや強めに設定。

10. **フッター**
    - ナビ（宿について / ご宿泊 / お知らせ / お食事 / アクセス / お問い合わせ）
    - 住所・Map・Instagram
    - コピーライト
    - スクロールの終端として、あまり動きを入れすぎない方針。

## 7. ディレクトリ設計（App Router 前提）

```txt
shiningresort-web/
├─ src/
│  ├─ app/
│  │  ├─ page.tsx              # トップページ（KOKON トップ再現のメイン）
│  │  ├─ about/
│  │  │  └─ page.tsx           # 宿について
│  │  ├─ stay/

│  │  │  └─ page.tsx           # 体験・散策
│  │  ├─ access/
│  │  │  └─ page.tsx           # アクセス
│  │  ├─ news/
│  │  │  └─ page.tsx           # お知らせ一覧
│  │  └─ layout.tsx            # 共通レイアウト（ヘッダー・フッター）
│  │
│  ├─ components/
│  │  ├─ layout/
│  │  │  ├─ Header.tsx
│  │  │  ├─ Footer.tsx
│  │  │  └─ MainLayout.tsx
│  │  ├─ home/
│  │  │  ├─ HeroSection.tsx
│  │  │  ├─ ConceptSection.tsx
│  │  │  ├─ NewsSection.tsx
│  │  │  ├─ RoomsSection.tsx
│  │  │  ├─ RestaurantSection.tsx
│  │  │  ├─ ExperienceSection.tsx
│  │  │  ├─ AccessSection.tsx
│  │  │  └─ ReservationSection.tsx
│  │  └─ common/
│  │     ├─ SectionTitle.tsx
│  │     ├─ Button.tsx
│  │     ├─ Card.tsx
│  │     ├─ Container.tsx
│  │     └─ MotionSection.tsx   # Framer Motion で共通のアニメーションラッパ
│  │
│  ├─ data/
│  │  ├─ news.ts           # News のダミーデータ（Phase1はハードコード）
│  │  ├─ rooms.ts          # 部屋データ（名称・説明・タグなど）
│  │  ├─ experiences.ts    # 体験・周辺スポット
│  │  └─ siteMeta.ts       # サイトタイトル、説明文など
│  │
│  ├─ hooks/               # カスタムフック（スクロール位置取得など）
│  ├─ lib/                 # 外部サービス連携や共通ロジック
│  ├─ utils/               # 純粋なユーティリティ関数
│  ├─ types/               # 型定義（Room, News など）
│  ├─ stores/              # 状態管理が必要になったときに利用（将来的に）
│  ├─ constants/           # カラーコード・ブレークポイント定数など
│  ├─ styles/
│  │  └─ globals.css       # Tailwind ベース + 全体スタイル調整
│  │
│  └─ ...
│
├─ public/
│  ├─ images/
│  │  ├─ hero/             # トップのヒーロー画像（仮 → 実写に差し替え）
│  │  ├─ rooms/
│  │  ├─ restaurant/
│  │  ├─ experience/
│  │  └─ common/
│  └─ favicon.ico
│
└─ package.json
```

- 最初は全部のディレクトリをフル活用する必要はない。
  - まずは `app/`, `components/`, `data/`, `styles/` だけでも良い。
  - 開発を進めながら、必要になったタイミングで `hooks/` や `types/` を増やしていく。
